1. Visualization Design Rationale

Our goal was to explore how global vegetation greenness (NDVI) changes across seasons. We combined a choropleth world map with a monthly trend chart to support spatial and temporal exploration.

Visual Encodings

  • Sequential green color scale for NDVI values.
  • Clean polygon borders for country shapes.
  • Line chart using position + connected lines to show seasonal patterns.

Data Transformations

  • Processed MODIS MOD13C2 NDVI data into country-level monthly averages.
  • Mapped each ISO3 code to a 12-month NDVI array.
  • Handled missing data and applied name/ISO corrections.

Interaction Design

  • Month slider for seasonal animation.
  • Hover tooltip for precise NDVI values.
  • Click a country to see a detailed monthly trend chart.
  • Pan + zoom for exploring regions closely.

2. Development Process

Overview of the Teamâ€™s Development Process

We first downloaded all twelve 2024 MOD13C2 HDF files from NASA Earthdata. After setting up a conda environment, we wrote a Python script to extract monthly NDVI, compute country-level averages, match ISO3 codes, and export a unified CSV. Final cleaning was completed in Jupyter Notebook.

With the dataset ready, we built the website and implemented core D3 components including the world map, color scale, month slider, pan/zoom, and the country trend chart. A major part of development involved debugging ISO mismatches in the TopoJSON.

Challenges & Solutions

  • Fixed unreliable ISO3 codes in TopoJSON using a custom name-to-ISO map and a better dataset.
  • Used console debugging to detect unmatched countries and repair mapping issues.

Time Spent

  • Data cleaning and preprocessing
  • Implementing D3 map, slider, and trend chart
  • Debugging ISO3 mismatches
  • Refining UI/UX
  • Creating checkpoint slides + recording and presenting the checkpoint video
